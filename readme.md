# arXiv AI 论文研究热点交互式可视化

大数据可视化与可视分析 · 期末课程设计

## 项目简介

本项目将 `arxiv_ai.csv` 中的 10,000 篇 arXiv AI 论文元数据组织成一个可交互探索的数据文档，聚焦 1993-2023 年人工智能研究热点的变化与主题关系，通过多视图联动展示热点演化、论文图谱、影响路径、机构样本和主题流派。

项目面向刚接触 AI 研究的学生或课程观众，帮助他们从大量论文中快速回答三个问题：

1. AI 领域有哪些细分方向或技术流派，它们如何随时间演进？
2. 众多论文之间如何形成主题、引用和结构关系？
3. 当聚焦于某一篇论文或一个主题时，如何继续找到相关论文、上游基础和下游延展？

## 数据说明

前端直接读取 `data/processed/` 中的静态 JSON：

| 文件 | 说明 | 当前规模 |
|---|---|---:|
| `nodes.json` | 论文节点，含标题、年份、作者、类别、主题、引用数、机构等字段 | 10,028 个节点 |
| `edges.json` | 论文引用关系，`source` 引用 `target` | 1,080 条边 |
| `keyword_trends.json` | 基于标题、摘要和类别规则抽取的年度主题热度 | 372 条记录 |
| `institutions_geo.json` | 可匹配机构的地理位置与影响力聚合 | 119 个机构 |
| `institution_aliases.json` | 机构名称归一化别名表 | 41 组 |
| `public/world.geojson` | 世界地图底图 | 1 份 |

主数据源是 `arxiv_ai.csv`。OpenAlex 缓存只用于补充引用数、真实引用边和机构字段，因此引用与机构视图应被理解为增强后的样本观察，而不是完整学术统计。

## 可视化模块

### 01 主题河流图

基于 arXiv AI 论文标题、摘要与类别字段的规则化主题标注，展示各研究方向年度论文数的河流式演化。支持年份范围筛选、主题聚焦、纵轴缩放和跨图表联动。

### 02 论文力导向图

将论文节点投射到关系网络中，揭示文献间的引用关系、主题聚类和局部邻域。支持论文/作者/主题搜索、缩放平移、节点拖拽、邻居高亮和详情查看。

### 03 蝴蝶影响图

选择一篇中心论文，左翼展示上游论文，右翼展示下游论文，用于追踪 1-3 阶局部影响路径。支持中心论文搜索、引用深度调节、年份筛选和点击切换中心节点。

### 04 机构影响力地图

展示可匹配论文背后的机构样本分布。支持地图缩放、颜色/大小维度切换、机构排行榜和详情面板。由于 arXiv 原始 CSV 不包含完整机构字段，本视图不应解释为完整全球机构排名。

### 05 主题流派地铁图

将时间线设为 X 轴，不同技术流派设为平行轨道，把跨主题论文抽象为换乘站。它用于解释 AI 研究方向如何分叉、汇合和再分化，是项目中较具新颖性的表达。

### 06 顶部关键节点时间线

页面顶部保留 AI/LLM 关键节点时间线作为背景参照。它不是 `arxiv_ai.csv` 主数据集的一部分，而是帮助观众把论文热点放入更广义 AI 发展脉络中理解。

## 全局联动

系统使用 `src/shared/app-state.js` 维护共享状态，包括年份范围、选中主题、选中论文和选中机构。任一视图中的筛选或选择会同步到其他视图和侧边栏，使用户可以按“主题总览 -> 论文结构 -> 局部路径 -> 机构样本 -> 流派交叉”的顺序探索。

## 技术栈

- 纯静态前端，ES Modules 直接加载
- 原生 SVG 可视化
- Leaflet 地图
- 自定义力导向布局与图分析交互
- 静态 JSON 数据管线
- EventTarget 全局状态同步

## 数据边界

- `arxiv_ai.csv` 是课程项目使用的 arXiv AI 样本，不代表完整 AI 学术数据库。
- 主题热度来自规则匹配，适合趋势观察，不等同于严格主题模型。
- 引用数、引用边、机构信息依赖 OpenAlex 匹配，覆盖率不完整。
- 机构地图是样本观察，不是全球机构贡献排名。
- 可视化主要服务于探索和理解，不应用作严肃文献计量结论。

## 运行方式

在项目根目录启动任意静态文件服务器，例如：

```powershell
python -m http.server 8000
```

然后访问本地服务器输出的地址。项目无需后端服务，可直接部署到 GitHub Pages。
