# LLM 论文发展脉络交互式数据文档

这是“大数据可视化与可视分析”课程项目。项目目标不是简单堆叠几张图，而是围绕 `data/seeds/seed.json` 中的核心论文，写成一篇可以交互探索的“LLM 论文发展数据文档”：从早期神经网络和表示学习，到 Transformer、预训练扩展、指令对齐、多模态、检索增强、长上下文、状态空间模型和智能体系统，观察论文、主题、引用关系与机构力量如何共同推动研究范式变化。

## 核心叙事

本文档的主线应以 seed 论文的发展顺序为骨架，而不是以图表模块本身为骨架。推荐的写作方式是：

1. 先用时间线式叙事提出问题：LLM 研究为什么会从“模型结构创新”转向“规模化预训练”，再转向“对齐、工具使用、多模态和智能体”？
2. 再把 seed 论文分成几个阶段，例如基础模型与表示学习、Transformer 与预训练、规模定律与大模型、开放模型与高效微调、对齐与偏好优化、RAG 与推理系统、多模态与长上下文、后 Transformer 架构探索。
3. 每个阶段挑 2-4 篇 seed 论文作为叙事锚点，说明它们解决了什么问题、继承了哪些前置工作、又影响了哪些后续方向。
4. 图表作为证据和探索工具嵌入文档：主题河流解释宏观趋势，论文网络解释结构位置，蝴蝶路径解释具体影响链，机构地图解释研究力量的空间和组织分布。

当前页面还没有完全达到这个理想形态。它已经具备数据和交互基础，但还需要把“图表展示”进一步组织成“读者沿 seed 论文主线推进的分析文章”。

## 当前数据

前端直接读取 `data/processed` 中的静态 JSON：

- `nodes.json`：论文节点，当前约 500 篇，包含 seed 论文和围绕 seed 扩展的引用/被引论文。
- `edges.json`：论文之间的引用关系，`source` 表示引用方，`target` 表示被引用方。
- `keyword_trends.json`：2013-2025 年 OpenAlex 年度主题热度，供主题河流图使用。
- `institutions_geo.json`：机构地理位置与聚合影响力。
- `public/world.geojson`：世界地图底图。

数据生成脚本：

```powershell
conda run -n base python scripts/build_processed_from_seed.py --target 500 --relation-limit 60 --offline-cache-only
conda run -n base python scripts/build_keyword_trends_openalex.py
```

说明：`seed.json` 是叙事核心，应严格保留其中论文字段。扩展论文用于补充引用网络和上下文，不应抢走 seed 论文在文档中的主角位置。

## 图表应如何使用

### 1. 主题河流图

主题河流图适合放在文章开头，用来回答“哪些研究方向在什么时候变热”。读者应该先通过它建立宏观时间感：

- 2013-2017：深度学习、NLP、计算机视觉、强化学习等基础方向持续增长。
- 2018-2020：Transformer、预训练语言模型、规模化训练成为中心。
- 2021-2023：大语言模型、开放模型、指令跟随、RAG、多模态开始快速上升。
- 2024-2025：LLM 应用、对齐、安全、智能体、长上下文和高效推理成为新的解释重点。

当前主题河流图相对可用，但仍需要在文档文字中解释 `count` 是 OpenAlex 原始论文量经过 log 缩放后的展示热度，而不是绝对论文数。

### 2. 论文力导向图

论文网络适合回答“某篇 seed 论文处在什么引用结构中”。建议用它辅助说明：

- 一篇论文是基础性论文、桥接论文，还是某个方向的后续应用论文。
- 它引用了哪些前置论文，又被哪些后续论文继续引用。
- 不同年份、不同主题之间是否存在跨阶段连接。

使用方式应该是：读者从文档文字中的一篇 seed 论文出发，点击或搜索该论文，在网络中查看它的一跳和两跳邻居。文档中应给出几个推荐探索入口，例如 BERT、GPT-3、InstructGPT、LLaMA、DPO、RAG、Mamba。

### 3. 蝴蝶路径图

当前蝴蝶路径图的显示效果很糟糕，主要问题是“它想表达影响路径，但读者不知道从哪里看起”。这个图不应该只是局部网络的另一种画法，而应该被改造成“论文之间的路径解释器”。

理想交互是：

- 左侧固定为前置论文或起点论文。
- 中间是桥接论文，例如 Transformer、BERT、GPT-3、InstructGPT、LLaMA。
- 右侧是后续论文或目标方向，例如 RAG、DPO、Mamba、Gemini、长上下文、多模态模型。
- 用户选择两篇 seed 论文后，图表显示它们之间最短引用路径或最强影响路径。
- 每条路径旁边提供一句解释：这条路径代表“架构继承”“训练范式扩展”“对齐方法演化”还是“系统化应用”。

下一步不建议继续优化当前杂乱的蝴蝶布局，而应重设计为“分层路径图”：

- 使用左中右三栏或 Sankey/Alluvial 布局。
- 只显示当前路径相关节点，不显示过多旁支。
- 节点标签必须始终可读，长标题要截断并在详情面板显示完整信息。
- 路径边需要带方向箭头，明确 `source cites target` 的语义。
- 增加预设故事路径，例如：
  - Word2Vec/BERT -> GPT -> GPT-3 -> InstructGPT
  - Transformer -> GPT-3 -> PaLM -> LLaMA
  - RAG -> tool use / agents
  - GAN/DDPM -> diffusion models -> multimodal generation
  - LLaMA -> QLoRA -> DPO -> open alignment

### 4. 机构地理图

当前机构地理图也不够好，主要问题是数据和表达都偏弱：许多扩展论文缺少机构字段，机构数量少，地理点过于稀疏，地图上的点无法自然回答“谁在推动这条论文主线”。

机构地图的角色应从“世界地图上有点”改成“研究力量分布说明”。它应该回答：

- 哪些机构在 seed 论文主线中出现频率最高？
- 公司实验室和高校分别在哪些阶段更活跃？
- 北美、欧洲、中国、日本等地区在不同阶段的贡献是否不同？
- 某篇 seed 论文背后对应哪些机构，以及这些机构还连接了哪些后续论文？

下一步优化方向：

- 补机构数据：优先为 seed 论文补齐 institution，再为高引用扩展论文补齐机构。
- 地图与论文选择联动：选中论文后，只高亮相关机构。
- 地图与主题联动：选择 “alignment” 或 “large language models” 后，只显示相关论文机构。
- 增加机构排行榜表格，与地图同步显示，避免地图点稀疏时信息不足。
- 对机构做归一化，例如 Google / Google Research / Google Brain / DeepMind / Google DeepMind 需要明确合并或拆分规则。
- 地图视觉上减少干扰，只保留点、少量连线和清晰 tooltip。

## 交互式数据文档的期望形态

最终成品应更接近一篇 scrollytelling 数据文章，而不是一个仪表盘。读者的阅读路径建议如下：

1. 开篇提出问题：LLM 论文发展为什么呈现阶段性跃迁？
2. 用主题河流图给出宏观背景，说明 2013-2025 的研究热点迁移。
3. 进入 seed 论文主线，按阶段讲述关键论文。
4. 每讲到一篇论文，就让论文网络自动聚焦到该论文，展示前置基础和后续影响。
5. 遇到两个阶段之间的连接时，用蝴蝶路径图解释“影响如何传递”。
6. 在每个阶段末尾，用机构地图说明“哪些机构在这一阶段形成主导力量”。
7. 结尾回到整体问题，归纳 LLM 研究从架构、数据、规模、对齐、系统和应用共同演化的路径。

换句话说，图表不是并列陈列，而是承担不同的证据功能：

- 主题河流图负责“趋势证据”。
- 论文网络负责“结构证据”。
- 蝴蝶路径图负责“因果/影响链解释”。
- 机构地图负责“空间与组织证据”。

## 下一步工作计划

### 第一阶段：修正文档主线

- 以 `seed.json` 论文为主线，整理一份阶段划分表。
- 每个阶段确定 2-4 篇必须讲的代表论文。
- 为每篇代表论文写一句“为什么重要”和一句“它连接到下一阶段的方式”。
- 页面文案要从“模块说明”改成“论文发展叙事”。

### 第二阶段：重做蝴蝶路径图

- 将当前局部网络布局替换为分层路径解释布局。
- 支持选择起点论文和终点论文。
- 默认提供 5-8 条预设论文路径。
- 只展示路径相关节点，减少噪声。
- 增加路径说明文本，让读者知道这条路径代表什么研究演化。

### 第三阶段：优化机构地理图

- 补齐 seed 论文机构字段。
- 统一机构名称，建立机构别名表。
- 增加机构排行榜和机构详情面板。
- 与论文网络联动：选中论文后高亮对应机构。
- 与主题河流联动：选中主题后显示该主题相关机构。

### 第四阶段：完善论文网络

- 默认只展示 seed 论文及其重要一跳邻居，避免 500 个节点同时造成阅读负担。
- 增加“只看 seed 主线”“看前置引用”“看后续影响”“看同主题论文”等模式。
- 节点样式区分 seed 论文、扩展论文、高引用论文、桥接论文。
- 详情面板提供摘要、作者、年份、引用数、相关路径入口。

### 第五阶段：补数据质量

- 对 seed 论文做人工校验，确保标题、年份、机构、主题、引用数可靠。
- 对扩展论文做去重和主题清洗，减少无关论文进入网络。
- 机构地理数据优先覆盖 seed 论文和高引用论文。
- 保留数据来源说明：Semantic Scholar 用于论文关系，OpenAlex 用于主题年度热度。

## 当前已知问题

- 蝴蝶路径图当前可读性不足，尚未真正承担“路径解释器”的功能。
- 机构地理图机构覆盖率不足，地图点和论文主线之间的关系不够明确。
- 部分论文机构字段为空，需要补齐。
- 当前页面文案仍偏“图表模块介绍”，还没有完全改写成 seed 论文发展主线。
- 主题河流图使用的是展示热度，不是绝对论文数，需要在页面中明确说明。

## 运行方式

在项目根目录启动任意静态文件服务器，例如：

```powershell
python -m http.server 8000
```

然后访问本地服务器输出的地址。项目无需后端服务，可直接部署到 GitHub Pages。
