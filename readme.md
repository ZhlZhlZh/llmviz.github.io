# LLM 论文研究热点可视化

这是“大数据可视化与可视分析”期末课程设计项目。项目以静态网页形式呈现，围绕 2013-2026 年大语言模型论文研究热点，提供可交互探索的数据文档。

## 项目问题

LLM 研究热点如何从表示学习与序列建模，演化到预训练扩张，再进入对齐、多模态与智能体阶段？哪些论文在引用网络中承担桥梁作用？研究机构影响力在地理和组织类型上呈现什么差异？

## 当前功能

- 主题河流图：展示关键词热度的年度变化，并将年份同步到其他图表，支持单主题聚焦。
- 论文力导向图：展示论文引用网络，支持年份过滤、节点选择、作者搜索与缩放拖拽。
- 蝴蝶影响路径图：以当前论文为中心展开局部网络，突出中心论文到目标论文的路径，支持全局/中心模式切换。
- 机构影响力地图：展示机构地理分布，并支持颜色/大小指标切换与机构联系线。
- AI 研究简史生成器：根据当前论文焦点生成阶段定位、前置基础与后续影响。
- 页面底部包含课程提交所需的设计说明、替代方案、数据引用和开发流程说明。

## 数据说明

前端直接读取 `data/processed` 中的静态 JSON 文件：

- `nodes.json`：论文节点。
- `edges.json`：引用关系。
- `keyword_trends.json`：关键词年度热度。
- `institutions_geo.json`：机构地理与影响力数据。
- `phases.json`：研究阶段划分。
- `nodes.csv`：论文节点的表格导出（便于检查）。
- `public/world.geojson`：世界地图底图。

当前数据为课程原型阶段整理的合成示例数据，用于压力测试交互和页面结构，不代表真实论文统计结果。若后续替换为真实论文语料，保持同一字段结构即可复用现有图表。

可用下面的脚本重新生成一批确定性的合成数据：

```powershell
node scripts/generate-fake-data.js
```

## 运行方式

在项目根目录启动任意静态文件服务器，例如：

```powershell
npx http-server .
```

也可以使用 Python：

```powershell
python -m http.server 8000
```

然后访问本地服务器输出的地址。项目无需后端服务，可直接部署到 GitHub Pages。
